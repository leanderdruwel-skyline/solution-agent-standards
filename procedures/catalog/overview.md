# Catalog Items - Overview

## Scope

These guidelines provide best practices for Catalog items published on docs.dataminer.services. This overview file links to specific validation procedures.

> **Note**: All files use severity tags `[ERROR]`, `[WARNING]`, and `[INFO]` as defined in [../global-instructions.md](../global-instructions.md#severity-levels).

---

## Validation Procedures

The following validation procedures must be followed for Catalog items:

### 1. Manifest Validation
See [manifest-validation.md](manifest-validation.md) for:
- Type definition
- Naming conventions
- Tags and metadata
- Owner configuration
- **Note**: Version information is excluded from validation as it's evaluated upon GitHub Release creation

### 2. Documentation Guidelines
See [documentation-validation.md](documentation-validation.md) for:
- README requirements for the `CatalogInformation` folder
- Documentation structure and content
- Best practices for describing functionality

### 3. GitHub Repository Validation
See [repository-validation.md](repository-validation.md) for:
- License requirements
- Repository naming
- .gitignore configuration
- GitHub topics
- Workflow links
- Dependabot configuration

### 4. README Validation
See [readme-validation.md](readme-validation.md) for:
- Repository README requirements
- Project README requirements
- Distinction from CatalogInformation README
